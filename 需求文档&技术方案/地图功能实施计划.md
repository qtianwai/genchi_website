# 地图功能实施计划

> 版本：v1.2  
> 更新日期：2026-04-04  
> 适用范围：地图页面第二轮迭代（顶部安全区、缩放阈值、聚合展开、卡片分层）

## 1. 本轮目标

在原有优化基础上，优先解决：
- 顶部不可点击与状态栏遮挡
- 地图缩放显示异常与聚合交互不稳定
- 点位视觉杂乱（状态叠加）
- 双指缩放时点位闪烁
- 店铺卡片按钮堆砌、主次不清

## 2. 本轮决策锁定

1. 模拟器回归设备：`iPhone 16 Plus`（当前环境无 `iPhone 15 Plus` 运行时）。
2. 缩放策略：平衡档三段式（店名/头像/聚合）+ 滞回防抖。
3. 卡片按钮策略：主次分层（主：收藏/避雷/导航；次：删除/分享）。

## 3. 阶段实施计划

### 阶段 1：顶部布局与可点击性修复（P0）
- 顶部操作区按安全区锚定：`top = safeAreaTop + 12`。
- 筛选/添加/定位按钮统一最小触控尺寸 `44pt`。
- 顶部控件层级高于地图层，避免地图手势抢占点击。
- 按钮保持中性无色样式，无选中高亮误导。

### 阶段 2：地图显示与交互 Bug 修复（P0）
- 缩放阈值三段式：
  - 店名显示：`latitudeDelta <= 0.06`
  - 仅头像：`0.06 < latitudeDelta < 0.11`
  - 聚合：`latitudeDelta >= 0.11`
- 滞回防抖：
  - 店名进入/退出：`<= 0.06` / `>= 0.065`
  - 聚合进入/退出：`>= 0.11` / `<= 0.105`
- 地图点位仅显示纯头像，移除收藏/避雷蒙版及角标。
- 聚合点击统一为：居中到簇中心并直接展开该簇店铺。
- 相机变化节流：仅在缩放桶变化、手势结束或节流窗口达到时重算点位。

### 阶段 3：店铺卡片重新设计（P0）
- 移除按钮区“视频源”按钮。
- 主次分层布局：
  - 主操作行：收藏、避雷、导航
  - 次操作行：删除、分享（占位）
- 保留实用功能：
  - 店铺图片放大预览
  - 店铺名称复制
  - 博主头像展示、博主名称复制、抖音链接跳转（保留在博主模块）
  - 删除防误触：标记删除→可取消→关闭卡片后生效
- 信息结构保持：店名、地址、美食分类、标签、推荐博主。

### 阶段 4：编译、调试与回归（P0）
- 构建方式：`Build iOS Apps` / `xcodebuild Debug`。
- 回归设备：`iPhone 16 Plus`（后续补 `iPhone 15 Plus`）。
- 核心验证：
  - 顶部按钮无遮挡且可点击
  - 缩放时先隐藏店名再聚合
  - 聚合展开居中且无异常跳级
  - 缩放过程点位无明显闪烁
  - 卡片主次清晰，复制/导航/抖音跳转/删除逻辑正常

## 4. 接口与改造范围

- 后端 API：无新增、无变更。
- iOS 侧改造：
  - `MapViewModel`：阈值常量、滞回判定、相机更新节流
  - `MapView`：顶部安全区与点击层级、纯头像点位、卡片按钮分层
- 文档改造：
  - `APP整体UI设计风格和要求.md`
  - `地图功能实施计划.md`

## 5. 验收标准

1. 顶部按钮在刘海屏无被状态栏遮挡，连续点击可触发。
2. 地图缩放遵循三段式阈值，显示切换稳定无抖动。
3. 点位仅显示纯头像，不再出现收藏/避雷叠加样式。
4. 点击聚合后地图居中并直接展开该簇。
5. 卡片按钮区为五项并具备主次层级，不拥挤。
6. 删除标记逻辑、复制、导航、抖音跳转均可用。
