# 用户信息自定义实施计划

> 创建时间：2026-04-03
> 功能：用户自定义昵称和头像，地图用户自建店铺标注显示用户头像

---

## 背景与目标

当前用户信息完全硬编码（昵称固定为"美食探索者"，无真实头像），地图上用户自建店铺标注使用通用紫色图标，无法体现个人身份。

本次目标：
1. 用户可自定义昵称（默认"美食探索者"）
2. 用户可上传头像（默认橙色占位符）
3. 地图用户自建店铺标注升级为显示用户自己的头像

---

## 一、数据库变更

### 1.1 新增 user_profiles 表

```sql
create table if not exists user_profiles (
  user_id     uuid primary key,
  nickname    text not null default '美食探索者',
  avatar_url  text,           -- null 表示未上传，使用默认占位符
  updated_at  timestamptz default now()
);

alter table user_profiles enable row level security;

-- profile 公开可读（地图标注异步加载头像需要）
create policy "profile 公开可读" on user_profiles
  for select using (true);

-- 用户只能修改自己的 profile（后端用 service_role key 绕过 RLS）
create policy "用户只能修改自己" on user_profiles
  for all using (auth.uid() = user_id);
```

### 1.2 Supabase Storage

在 Supabase 控制台创建 `avatars` bucket：
- 访问类型：**Public**（头像需公开可读）
- 文件路径规则：`{user_id}.jpg`（固定路径，上传新头像直接覆盖）
- 文件大小限制：2MB（后端校验）

上传后公开 URL 格式：
```
https://ygsxhvsmivcckmjmjmhr.supabase.co/storage/v1/object/public/avatars/{user_id}.jpg
```

## 二、后端变更

### 2.1 db.py 新增函数

**文件：`backend/db.py`**

```python
def get_user_profile(user_id: str) -> dict | None:
    """获取用户 profile，不存在返回 None"""
    result = supabase.table("user_profiles").select("*").eq("user_id", user_id).execute()
    return result.data[0] if result.data else None

def upsert_user_profile(user_id: str, nickname: str = None, avatar_url: str = None) -> dict:
    """创建或更新用户 profile（upsert）"""
    data = {"user_id": user_id, "updated_at": "now()"}
    if nickname is not None:
        data["nickname"] = nickname
    if avatar_url is not None:
        data["avatar_url"] = avatar_url
    result = supabase.table("user_profiles").upsert(
        data, on_conflict="user_id"
    ).execute()
    return result.data[0] if result.data else {}
```

### 2.2 main.py 新增接口

**文件：`backend/main.py`**

新增三个接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/profile/{user_id}` | GET | 获取用户 profile，不存在返回默认值 |
| `/api/profile/update` | POST | 更新昵称，长度校验 1-20 字符 |
| `/api/profile/upload-avatar` | POST | 上传头像（multipart/form-data），校验类型和大小 |

### 3.1 Models.swift

### 3.2 AuthState.swift

新增字段：
- `@Published var nickname: String = "美食探索者"`
- `@Published var avatarURL: String? = nil`

新增/修改方法：
- `checkStoredSession()` — 恢复登录态后调用 `loadProfile()`
- `verifyOTP()` — 登录成功后调用 `loadProfile()`
- `signOut()` — 重置 nickname 和 avatarURL
- 新增 `loadProfile(userId:)` — 调用 GET /api/profile/{user_id}
- 新增 `updateNickname(_ nickname:)` — 调用 POST /api/profile/update
- 新增 `uploadAvatar(_ imageData:)` — multipart 上传头像

### 3.3 ProfileView.swift

改造要点：
- **头像区域**：`AsyncImage` 加载 avatarURL；无头像时显示橙色圆形 + `person.fill` 占位符；右下角铅笔角标，点击触发 `PhotosPicker`
- **昵称区域**：显示 `authState.nickname`，旁边铅笔图标，点击弹出 Alert 输入框修改
- 使用 `PhotosPicker`（iOS 16+，无需第三方库）

### 3.4 MapView.swift — UserPinView 改造

`UserPinView`（当前第132-153行）新增参数 `avatarURL: String?`：
- **有头像**：`AsyncImage` 加载，圆形裁剪，风格与 `MapPinView` 一致
- **无头像**：保持原有紫色 `person.crop.circle.badge.plus` 图标

调用处（MapView 中 ForEach filteredUserRestaurants）传入 `authState.avatarURL`。

## 四、默认头像规则

| 状态 | 显示效果 |
|------|---------|
| `avatar_url == nil` | 橙色圆形 + `person.fill` 图标 |
| `AsyncImage` 加载中 | 同上占位符 |
| `AsyncImage` 加载失败 | 同上占位符 |
| 已上传头像 | 圆形裁剪的用户头像 |

ProfileView 和 UserPinView 共用同一套占位符逻辑。

## 六、验证方式

1. 登录后进入个人页，昵称默认"美食探索者"，头像显示橙色占位符
2. 点击头像 → 选择相册图片 → 上传成功后头像实时更新
3. 点击昵称旁铅笔图标 → 输入新昵称 → 确认后立即生效
4. 退出登录再重新登录，昵称和头像仍保持（从服务端加载）
5. 地图切换"我的推荐"，已上传头像的用户自建店铺标注显示用户头像；未上传时仍显示紫色图标
