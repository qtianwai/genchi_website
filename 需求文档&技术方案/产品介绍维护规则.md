# 产品介绍文档维护规则

## 规则目的

`产品介绍.md` 是面向用户的核心文档，让不了解产品的人能快速上手使用。当产品功能发生变化时，必须同步更新该文档。

## 维护时机

当涉及以下变更时，必须检查并更新 `产品介绍.md`：

| 变更类型 | 是否需要更新文档 | 更新内容示例 |
|---------|----------------|-------------|
| 新功能上线 | ✅ 必须更新 | 在「核心功能」新增功能说明 |
| 功能调整/优化 | ⚠️ 视情况 | 若影响用户体验方式，需同步更新 |
| 功能下线 | ✅ 必须更新 | 从「核心功能」移除或注明已停用 |
| 使用流程变化 | ✅ 必须更新 | 更新相关功能描述或 FAQ |
| 常见问题解答 | ⚠️ 视情况 | 新增/修改 FAQ |

## 更新原则

1. **保持结构稳定**：仅在现有结构上补充或修正，不随意调整文档框架
2. **语言简洁直观**：
   - 面向普通用户，避免技术术语
   - 用「你」称呼用户
   - 每项功能一句话说清楚价值
3. **与产品功能清单保持一致**：新增功能应同步更新 `产品功能清单.md`

## 更新检查清单

每次更新时确认以下内容：

- [ ] 新功能是否在「核心功能」中有体现
- [ ] 功能名称和描述是否与实际 UI 一致
- [ ] FAQ 是否需要新增或修改
- [ ] 「产品特色」是否需要补充
- [ ] 日期是否已更新

## 文档结构（勿轻易调整）

```
产品介绍.md
├── 产品定位          # 一句话说清产品是什么
├── 核心功能          # 主要功能列表，每个功能一段
├── 使用场景          # 典型使用情境
├── 产品特色          # 区别于其他产品的亮点
├── 常见问题          # FAQ
└── 更新日期          # 每次更新需修改
```

## 责任人

产品功能相关的任何调整，由开发者在完成开发时同步更新此文档。
