# 手动输入店铺功能设计

## 一、功能概述

当 AI 无法从视频中识别出店铺时，允许用户手动输入店铺名称和城市，系统通过高德地图搜索并入库。

## 二、触发场景

1. `/api/parse-link` 返回 `status=not_found`（未识别到店铺）
2. 前端显示"未能识别到具体店铺"提示
3. 显示"手动添加店铺"按钮

## 三、交互流程

```
用户点击"手动添加店铺"
       │
       ▼
弹出输入表单
  ├─ 店铺名称（必填）
  ├─ 所在城市（必填，下拉选择）
  └─ 美食分类（可选，如：烤肉、火锅）
       │
       ▼
用户提交 → 调用 /api/manual-add-restaurant
       │
       ├─ 高德地图搜索坐标
       │   ├─ 找到 → 入库 + 关联博主 + 更新视频缓存
       │   └─ 未找到 → 提示"高德地图未找到该店铺，请检查名称"
       │
       └─ 返回结果 → 刷新地图
```

## 四、API 设计

### 请求接口

```
POST /api/manual-add-restaurant
```

### 请求参数

```json
{
  "video_url": "https://v.douyin.com/xxx/",  // 原始视频链接
  "user_id": "uuid",
  "restaurant_name": "上海新Q烤吧",
  "city": "上海",
  "category": "烤肉"  // 可选
}
```

### 响应格式

```json
{
  "status": "success",
  "restaurant": {
    "id": "uuid",
    "name": "上海新Q烤吧（斜土路店）",
    "address": "斜土路xxx号",
    "city": "上海",
    "latitude": 31.xxx,
    "longitude": 121.xxx,
    "category": "烤肉"
  },
  "message": "店铺已添加到地图"
}
```

### 错误响应

```json
{
  "status": "error",
  "message": "高德地图未找到该店铺，请检查店铺名称和城市"
}
```

## 五、数据库操作

1. 查询 `video_parse_cache` 表，获取 `video_id` 和 `author_id`
2. 调用高德地图搜索 API（`batch_search_restaurants`）
3. 入库 `restaurants` 表（如果不存在）
4. 关联 `author_restaurants` 表（`link_author_restaurant`）
5. 更新 `video_parse_cache` 表状态为 `completed`

## 六、前端 UI 设计

### ParseLinkSheet.swift 修改

1. 当 `result.status == "not_found"` 时，显示"手动添加店铺"按钮
2. 点击按钮弹出 `ManualAddRestaurantSheet`
3. 用户填写表单并提交
4. 提交成功后刷新地图并关闭弹窗

## 七、优势

1. **提升数据完整性**：AI 识别失败时，用户可以手动补充
2. **帮助其他用户**：手动添加的店铺会被缓存，其他用户粘贴相同视频时直接命中
3. **降低流失率**：避免用户因识别失败而放弃使用
4. **数据质量保证**：通过高德地图验证，确保店铺真实存在且有坐标

## 八、注意事项

1. **防止滥用**：同一用户对同一视频只能手动添加一次
2. **数据验证**：店铺名称长度限制（2-50 字符），城市必须从预设列表选择
3. **高德搜索失败处理**：提示用户检查店铺名称，或提供"跳过"选项
4. **缓存更新**：手动添加成功后，更新 `video_parse_cache` 表，避免重复解析
