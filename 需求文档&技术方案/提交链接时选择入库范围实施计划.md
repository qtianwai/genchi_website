## 二、后端改动（`backend/main.py`）

### 2.1 扩展 `ParseLinkRequest` 模型

```python
class ParseLinkRequest(BaseModel):
    url: str        # 抖音分享链接
    user_id: str    # 当前用户 ID
    scope: str = "follow_all"  # 入库范围：
                               # "follow_all"  → 关注博主 + 触发后台全量解析（默认，保持现有行为）
                               # "single_only" → 仅添加本店铺，不触发后台任务，不自动关注博主
```

### 2.2 修改 `parse_link` 接口逻辑

在第十步（自动关注博主）和第十二步（启动后台任务）处，增加 `scope` 判断：

**第十步（约 main.py:855）：**
```python
# 第十步：自动关注博主（仅 follow_all 模式）
if req.scope != "single_only":
    follow_author(req.user_id, author_id)
```

**第十一步（自动更新检测，约 main.py:858）：**
```python
# 第十一步：启用/重新激活博主的自动更新检测（仅 follow_all 模式）
if req.scope != "single_only":
    # ... 现有逻辑不变 ...
```

**第十二步（启动后台任务，约 main.py:879）：**
```python
# 第十二步：启动后台任务（仅 follow_all 模式）
is_bg_running = False
if req.scope != "single_only":
    # ... 现有冷却期检查 + background_tasks.add_task 逻辑不变 ...
```

### 2.3 `single_only` 模式的行为差异

| 行为 | `follow_all`（默认） | `single_only` |
|------|---------------------|---------------|
| 解析当前视频 | ✅ | ✅ |
| 店铺入库 | ✅ | ✅ |
| 自动关注博主 | ✅ | ❌ |
| 触发后台历史视频解析 | ✅ | ❌ |
| 启用博主自动更新检测 | ✅ | ❌ |
| 返回 `is_background_running` | 可能为 true | 始终为 false |

## 三、iOS 改动

### 3.1 `APIService.swift`（约第 65 行）

扩展 `parseDouyinLink` 方法签名，新增 `scope` 参数：

```swift
func parseDouyinLink(url: String, userId: String, scope: String = "follow_all") async throws -> ParseLinkResponse {
    struct Body: Codable { let url: String; let user_id: String; let scope: String }
    return try await post(
        path: "/api/parse-link",
        body: Body(url: url, user_id: userId, scope: scope)
    )
}
```

### 3.2 `ParseLinkSheet.swift` — 新增选择 UI

**新增状态变量：**
```swift
@State private var selectedScope: ParseScope = .followAll

enum ParseScope {
    case followAll   // 关注博主全部推荐
    case singleOnly  // 仅添加本店铺
}
```

**UI 位置：** 在链接输入框下方、解析按钮上方，插入一个 Picker 或两个选项卡按钮：

```
┌─────────────────────────────────────────┐
│  粘贴抖音链接...                          │
└─────────────────────────────────────────┘

  ○ 关注博主全部推荐（推荐）
    解析博主所有探店视频，自动关注博主
  ● 仅添加本店铺
    只添加这条视频的店铺，不关注博主

[        开始解析        ]
```

UI 组件选用两个可点击的选项行（RadioButton 风格），视觉上参考现有 DesignSystem 样式。

**修改 `parseLink()` 函数：**
```swift
let response = try await APIService.shared.parseDouyinLink(
    url: linkText.trimmingCharacters(in: .whitespaces),
    userId: authState.userId,
    scope: selectedScope == .singleOnly ? "single_only" : "follow_all"
)
```

**`single_only` 模式下，不启动后台进度轮询**（`is_background_running` 后端已返回 false，现有判断逻辑自然跳过，无需额外处理）。

## 六、注意事项

- `scope` 默认值为 `"follow_all"`，保持现有行为完全不变，向后兼容。
- `single_only` 模式下，博主记录仍会 upsert（`upsert_author` 步骤不变），只是不建立 follow 关系、不触发后台任务。
- 不需要新增数据库字段，`data_source` 字段已有 `user_submit` 可区分，无需额外标记。
