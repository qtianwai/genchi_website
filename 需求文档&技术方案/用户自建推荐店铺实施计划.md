# 用户自建推荐店铺 — 实施计划

> 版本：v1.0 | 日期：2026-04-03 | 优先级：P1

## 背景与目标

当前所有店铺数据都来自「解析抖音视频 → AI 识别 → 关联博主」这条链路，用户无法独立添加自己知道的好店。本功能让用户脱离博主依赖，直接输入店名/城市搜索高德 POI 后入库，标记为「我的推荐」并在地图上区分展示。

这也是「个人专属美食地图」（P2）的前置依赖。

---

## 产品设计

### 用户路径

1. 底部 Tab 栏居中位置新增「+」按钮（仿抖音风格，突出显示）
2. 点击「+」→ 弹出选择面板，提供两个选项：
   - 「解析抖音链接」→ 原有 `ParseLinkSheet` 流程（不变）
   - 「手动添加店铺」→ 新的 `UserAddRestaurantSheet`（两步流程）
     - Step 1：输入店名 + 选择城市 → 点击搜索 → 显示高德候选列表（最多 5 条）
     - Step 2：点击候选确认 → 入库 → 地图刷新，出现「我的推荐」标注

### 地图区分展示

| 类型 | 标注样式 |
|------|---------|
| 博主推荐 | 圆形博主头像（橙色，现有） |
| 我的推荐 | 紫色圆形 + `person.crop.circle.badge.plus` 图标 |

顶部筛选栏新增「我的推荐」chip（紫色图标），排在「全部」之后。

### 管理入口

复用「收藏」Tab（`FavoritesView`），页面顶部新增分段控制器切换：
- 「我的收藏」：现有收藏列表（不变）
- 「我的推荐」：用户自建店铺列表，支持滑动删除

删除逻辑：只删 `user_created_restaurants` 记录，不动 `restaurants` 表（其他用户可能也在用）。

### 边界情况

同一家店（同 `amap_id`）既被博主推荐又被用户自建时：
- 地图只显示一个标注，**优先用户头像样式**
- 店铺详情卡片底部显示关联的博主推荐信息

## 技术方案

### 1. 数据库

新建 `user_created_restaurants` 表（不改现有表，`author_restaurants` 语义不兼容）：

```sql
create table if not exists user_created_restaurants (
  id              uuid primary key default uuid_generate_v4(),
  user_id         uuid not null,
  restaurant_id   uuid not null references restaurants(id) on delete cascade,
  note            text,                          -- 预留备注字段
  created_at      timestamptz default now(),
  unique(user_id, restaurant_id)                 -- 同一用户不能重复添加同一家店
);
create index if not exists idx_ucr_user on user_created_restaurants(user_id);
create index if not exists idx_ucr_restaurant on user_created_restaurants(restaurant_id);
```

`restaurants` 表无需改动，`amap_id` 唯一键天然去重。

### 2. 后端 API

新建独立路由（不扩展 `manual-add-restaurant`，职责不同）：

| 接口 | 说明 |
|------|------|
| `GET /api/user-restaurants/search?name=&city=` | 复用高德搜索，返回最多 5 条候选 |
| `POST /api/user-restaurants` | upsert restaurants（按 amap_id）→ insert user_created_restaurants |
| `GET /api/user-restaurants?user_id=` | 查询用户自建列表，join restaurants |
| `DELETE /api/user-restaurants/{restaurant_id}?user_id=` | 只删 user_created_restaurants 记录 |

`/api/map/restaurants` 响应新增 `user_restaurants` 字段（向后兼容，保留原有字段）：
```python
return {
    "restaurants": author_data,        # 原有，不变
    "user_restaurants": user_data      # 新增
}
```

关键文件：`backend/main.py`、`backend/db.py`

### 3. iOS 端

> 前端开发使用 `frontend-design` skill 把控设计质量，确保 UI 风格与现有 DesignSystem 一致。

**新增 Model（`Models.swift`）：**

struct UserCreatedRestaurant: Identifiable, Codable {

**`MainTabView.swift`：** 底部居中新增「+」按钮，点击弹出选择面板；原 `MapView` 内右下角「粘贴链接」按钮移除。

**`MapView.swift` + `MapViewModel`：** 新增用户标注渲染（`UserMapPinView`）、「我的推荐」筛选 chip。

**新建文件：** `UserAddRestaurantSheet.swift`（两步表单：搜索候选 → 确认入库）

**`APIService.swift` 新增方法：**
- `searchUserRestaurant(name:city:)`
- `createUserRestaurant(...)`
- `getUserRestaurants(userId:)`

## 验证方式

1. 数据库：查询 `user_created_restaurants` 表结构确认建表成功
2. 后端：`curl /api/user-restaurants/search?name=海底捞&city=上海` 返回候选列表
3. 后端：POST 创建记录，GET 查询确认返回
4. iOS：点击「+」弹出选择面板，选「手动添加」后搜索并添加，地图出现紫色标注
5. 删除：滑动删除后地图标注消失，`restaurants` 表记录仍存在
