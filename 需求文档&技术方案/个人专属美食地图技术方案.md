# 个人专属美食地图 — 技术方案（v6.0）

## 产品定位

用户可以将自己关注的博主推荐 + 自建推荐整合为一张可分享的个人地图，其他用户可通过分享链接查看并一键订阅，订阅后在自己的地图上能看到对方推荐的店铺（有来源标记，可独立开关）。

---

## 核心设计原则

- 订阅用户地图数据**不在后端合并**，iOS 端按需并发加载多个数据源
- 订阅关系独立于「关注博主」，是独立的「用户地图订阅」系统
- 点位来源清晰标记（谁推荐的），支持按订阅用户筛选
- 基础隐私控制（公开/私密），私密地图后端拦截

---

## 数据库设计

### 新增表：user_maps（用户地图表）

```sql
CREATE TABLE IF NOT EXISTS user_maps (
  user_id    uuid PRIMARY KEY,
  is_public  boolean NOT NULL DEFAULT true,
  updated_at timestamptz DEFAULT now()
);
```

**字段说明：**
- `user_id`：用户 ID，对应 `user_profiles.user_id`
- `is_public`：是否公开（true=公开，false=私密）
- `updated_at`：最后更新时间

**RLS 策略：**
- 公开地图任何人可读
- 私密地图只有本人可读
- 用户只能修改自己的地图设置

**初始化：**
为现有用户初始化地图记录，默认公开。

### 新增表：user_map_subscriptions（用户地图订阅表）

```sql
CREATE TABLE IF NOT EXISTS user_map_subscriptions (
  id              uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  subscriber_id   uuid NOT NULL,
  target_user_id  uuid NOT NULL,
  is_enabled      boolean NOT NULL DEFAULT true,
  created_at      timestamptz DEFAULT now(),
  UNIQUE(subscriber_id, target_user_id)
);
```

**字段说明：**
- `subscriber_id`：订阅者（看别人地图的人）
- `target_user_id`：被订阅者（地图所有者）
- `is_enabled`：是否在自己地图上显示该用户点位（开关）
- `created_at`：订阅时间

**RLS 策略：**
- 用户只能操作自己的订阅

## 后端 API 设计

### 用户地图相关接口

#### GET /map/{user_id}
H5 预览页（降级方案）。未安装 App 时显示此页面，已安装时 Universal Link 跳转 App。

**返回：** HTML 页面，包含 Smart App Banner 和店铺列表文字版

**隐私处理：** 私密地图返回「该地图已设为私密」提示页

---

#### GET /api/map/{user_id}/info
获取他人地图基本信息。

**参数：** 无

**返回：**
```json
{
  "user_id": "uuid",
  "nickname": "string",
  "avatar_url": "string or null",
  "is_public": boolean,
  "restaurant_count": integer
}
```

**隐私处理：** 私密地图返回 `{ is_private: true }`

---

#### GET /api/map/{user_id}/restaurants
分页获取他人地图的店铺列表。

**参数：**
- `page`：页码（默认 1）
- `page_size`：每页数量（默认 50）
- `lat`：纬度（可选，用于附近筛选）
- `lng`：经度（可选，用于附近筛选）
- `radius_km`：搜索半径（可选，默认 10km）

**返回：**
```json
{
  "is_private": false,
  "restaurants": [
    {
      "id": "string",
      "restaurant_id": "string",
      "name": "string",
      "address": "string",
      "city": "string",
      "category": "string",
      "latitude": number,
      "longitude": number,
      "photo_url": "string or null"
    }
  ],
  "total": integer,
  "has_more": boolean
}
```

**隐私处理：** 私密地图返回 `{ is_private: true, restaurants: [] }`

---

#### POST /api/map/privacy
更新自己地图的公开/私密设置。

**参数：**
```json
{
  "user_id": "string",
  "is_public": boolean
}
```

**返回：** `{ status: "ok" }`

**副作用：** 切换为私密后，所有订阅者后续无法访问该地图数据

---

### 地图订阅相关接口

#### GET /api/map-subscriptions?user_id=
获取我的订阅列表。

**返回：**
```json
{
  "subscriptions": [
    {
      "id": "uuid",
      "target_user_id": "uuid",
      "nickname": "string",
      "avatar_url": "string or null",
      "is_enabled": boolean,
      "created_at": "timestamp"
    }
  ]
}
```

---

#### POST /api/map-subscriptions
订阅他人地图。

**参数：**
```json
{
  "subscriber_id": "string",
  "target_user_id": "string"
}
```

**返回：** `{ status: "ok" }`

**校验：**
- `subscriber_id ≠ target_user_id`（不能订阅自己）
- 检查 `user_maps.is_public`（私密地图无法订阅）

---

#### DELETE /api/map-subscriptions/{target_user_id}?subscriber_id=
取消订阅。

**返回：** `{ status: "ok" }`

---

#### PATCH /api/map-subscriptions/{target_user_id}
切换订阅开关（是否在自己地图上显示该用户点位）。

**参数：**
```json
{
  "subscriber_id": "string",
  "is_enabled": boolean
}
```

**返回：** `{ status: "ok" }`

## iOS 实现设计

### 数据模型扩展

#### RecommendSourceType（推荐来源）
```swift
enum RecommendSourceType {
    case author(Author)
    case selfCreated
    case subscribedUser(userId: String, nickname: String, avatarUrl: String?)
}
```

#### MapDisplayItem 扩展
```swift
struct MapDisplayItem: Identifiable {
    // ... 现有字段 ...
    let recommendedBy: [RecommendSourceType]  // 所有推荐来源
}
```

#### 订阅相关模型
```swift
struct UserMapInfo: Codable {
    let user_id: String
    let nickname: String
    let avatar_url: String?
    let is_public: Bool
    let restaurant_count: Int
}

struct MapSubscription: Codable, Identifiable {
    let id: String
    let target_user_id: String
    let nickname: String
    let avatar_url: String?
    var is_enabled: Bool
    let created_at: String?
}

struct UserMapRestaurantItem: Codable, Identifiable {
    let id: String
    let restaurant_id: String
    let name: String
    let address: String?
    let city: String?
    let category: String?
    let latitude: Double?
    let longitude: Double?
    let photo_url: String?
}
```

### MapViewModel 扩展

**新增属性：**
```swift
@Published var subscribedMapData: [String: [UserMapRestaurantItem]] = [:]
@Published var mapSubscriptions: [MapSubscription] = []
```

**并发加载限制：** 最多同时加载 3 个订阅用户数据，超出的串行等待

**双重去重逻辑：**
1. 按 `restaurant_id` 去重
2. 坐标 50m 内同名店铺兜底合并
3. 信息优先级：自建 > 订阅用户 > 博主推荐

**订阅状态同步：** App `onAppear` 时重新拉取订阅列表，保证多端一致

## 一级漏洞修复清单

### 1. 并发请求雪崩
**修复：** 并发加载限制 3 个 + 按定位懒加载

### 2. 去重仅依赖 restaurant_id
**修复：** 双重去重（restaurant_id + 坐标 50m 内同名兜底）+ 信息优先级

### 3. 隐私权限边界漏洞
**修复：** 
- H5 页同步校验 `is_public`
- 公开→私密切换后自动屏蔽订阅者数据访问

### 4. 无「订阅自己」拦截
**修复：** 后端订阅接口增加校验 `subscriber_id ≠ target_user_id`

### 5. 乐观更新无兜底
**修复：** 
- Toggle 切换失败回滚
- App `onAppear` 时重新拉取订阅列表
