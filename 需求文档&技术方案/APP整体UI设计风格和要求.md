# APP整体UI设计风格和要求

> 版本：v1.1  
> 日期：2026-04-04  
> 适用范围：FoodMap iOS 全页面（地图、收藏、详情、解析、个人中心）

## 1. 设计目标

- 保持“轻、清晰、可快速操作”的视觉体验。
- 强化信息优先级：先看核心信息，再看辅助信息。
- 保证高频操作一步可达，避免误触与歧义。
- 全局统一浅色风格，减少视觉跳变。

## 2. 全局视觉规范

### 2.1 主题与色彩
- 全局固定浅色主题（Light Only），不跟随系统暗色。
- 品牌主色：橙色（用于主行动、强调态）。
- 语义色：
  - 成功：绿色
  - 危险：红色
  - 信息：蓝色
- 默认操作按钮优先使用中性底色，避免“未操作即高亮”。

### 2.2 设计 Token（与 `DesignSystem.swift` 对齐）
- 间距：`4 / 8 / 12 / 16 / 20 / 24`
- 圆角：`8 / 12 / 16 / 20`
- 阴影：卡片轻阴影，不使用重投影
- 分割线：低对比、辅助层级，不抢主内容

### 2.3 排版规范
- 页面标题：`Title2` 或 `Headline`（按页面层级）
- 主信息：`Body / Subheadline`，保证可读性
- 辅助信息：`Caption / Caption2`
- 一屏中字号层级建议不超过 4 级

## 3. 组件级规范

### 3.1 顶部工具条与安全区
- 顶部操作区必须锚定安全区，基线：`top = safeAreaTop + 12`。
- 顶部工具按钮最小触控尺寸：`44pt x 44pt`。
- 左侧筛选、右侧主操作（如添加）保持同基线。
- 顶部控件层级高于地图手势层，保证点击命中稳定。

### 3.2 地图点位与缩放阈值
- 地图点位仅显示纯头像，不叠加收藏/避雷角标或蒙版。
- 缩放三段式规则（平衡档）：
  - 店名显示：`latitudeDelta <= 0.06`
  - 仅头像：`0.06 < latitudeDelta < 0.11`
  - 聚合：`latitudeDelta >= 0.11`
- 阈值滞回防抖：
  - 店名进入/退出：`<= 0.06` / `>= 0.065`
  - 聚合进入/退出：`>= 0.11` / `<= 0.105`
- 聚合点击行为：地图居中到簇中心并直接展开该簇店铺。

### 3.3 搜索模块
- 搜索框与工具区同视觉体系。
- 空输入展示近期搜索历史。
- 输入后展示模糊建议，文案优先，样式简洁。

### 3.4 店铺卡片
- 信息结构固定：店名、地址、分类/标签、推荐博主。
- 操作区主次分层：
  - 主操作：收藏、避雷、导航
  - 次操作：删除、分享（占位）
- 抖音跳转保留在博主模块，不放在按钮区。
- 危险操作采用“可撤销/二次生效”策略（删除标记）。
- 图片可点击放大，复制操作需即时反馈。

## 4. 交互反馈规范

- 点击反馈：`0.95~0.98` 缩放 + 轻微透明度变化。
- 文本复制成功后提供轻提示（Toast）。
- 缩放过程中避免点位频繁重建闪烁。
- 异步操作失败需给出可理解提示，不静默失败。

## 5. 适配与可用性规范

- 适配范围至少覆盖：刘海屏、非刘海屏、大屏（含 iPhone 15 Plus）。
- 地图缩放、拖动、点按时不出现遮挡和布局抖动。
- 底部卡片与系统手势区保持安全间距。
- 功能入口不得被地图手势层吞掉点击事件。

## 6. 开发执行要求

- 新增样式优先复用 `DS` Token，不新增散落硬编码。
- 页面改造需附带对应交互回归验证。
- 涉及导航、复制、外链跳转的能力必须在真机/模拟器验证可触发。
- 竞品可参考信息结构，不得照搬视觉细节。

## 7. 验收清单（UI）

1. 全局浅色主题稳定生效。
2. 顶部按钮无状态栏遮挡、连续点击可触发。
3. 地图缩放符合三段阈值，且缩放过程无明显闪烁。
4. 聚合点击后居中并直接展开。
5. 卡片主次层级清晰，五项按钮触发正常。
6. 复制、导航、抖音跳转、删除标记流程正常。
7. 多机型布局无错位/遮挡。
