# 收藏模块功能调整与完善 — 实施计划

> 版本：v1.0 | 日期：2026-04-04 | 优先级：P1

## 背景与目标

当前 App 的「博主」和「收藏」分属两个独立 Tab，功能割裂。用户需要在不同页面间切换才能管理关注的博主和收藏的店铺。本次重构将两者合并为统一的「收藏」入口，同时新增博主详情页、店铺详情全屏页、店铺列表分组管理页，并引入避雷、删除、收藏理由等精细化数据操作，提升整体用户体验。

**关键约束：**
- "拉黑"概念全部改为"避雷"，前端所有显示文案均用"避雷"
- 均价字段不展示（目前拿不到数据）
- 地图筛选栏中"我的推荐"筛选 chip 需使用用户真实头像（非 icon）
- 收藏页店铺行：左滑操作为「收藏/取消收藏」「避雷」「删除」
- 收藏理由入口放在店铺卡片上（💬留言小图标），不放在左滑菜单；卡片上同时有📍导航图标
- 一个店铺只维护一条用户的收藏理由

## 一、数据库变更

### 新增表（4张）

```sql
-- 1. 用户避雷店铺表
CREATE TABLE IF NOT EXISTS user_blocked_restaurants (
  id              uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id         uuid NOT NULL,
  restaurant_id   uuid NOT NULL REFERENCES restaurants(id) ON DELETE CASCADE,
  created_at      timestamptz DEFAULT now(),
  UNIQUE(user_id, restaurant_id)
);
CREATE INDEX IF NOT EXISTS idx_ubr_user ON user_blocked_restaurants(user_id);
ALTER TABLE user_blocked_restaurants ENABLE ROW LEVEL SECURITY;
CREATE POLICY "用户只能操作自己的避雷记录" ON user_blocked_restaurants FOR ALL USING (auth.uid() = user_id);

-- 2. 用户删除店铺表（全局隐藏）
CREATE TABLE IF NOT EXISTS user_deleted_restaurants (
  id              uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id         uuid NOT NULL,
  restaurant_id   uuid NOT NULL REFERENCES restaurants(id) ON DELETE CASCADE,
  created_at      timestamptz DEFAULT now(),
  UNIQUE(user_id, restaurant_id)
);
CREATE INDEX IF NOT EXISTS idx_udr_user ON user_deleted_restaurants(user_id);
ALTER TABLE user_deleted_restaurants ENABLE ROW LEVEL SECURITY;
CREATE POLICY "用户只能操作自己的删除记录" ON user_deleted_restaurants FOR ALL USING (auth.uid() = user_id);

-- 3. 用户自定义分组表
CREATE TABLE IF NOT EXISTS user_restaurant_groups (
  id          uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id     uuid NOT NULL,
  name        text NOT NULL,
  created_at  timestamptz DEFAULT now(),
  UNIQUE(user_id, name)
);
CREATE INDEX IF NOT EXISTS idx_urg_user ON user_restaurant_groups(user_id);
ALTER TABLE user_restaurant_groups ENABLE ROW LEVEL SECURITY;
CREATE POLICY "用户只能操作自己的分组" ON user_restaurant_groups FOR ALL USING (auth.uid() = user_id);

-- 4. 分组-店铺关联表
CREATE TABLE IF NOT EXISTS user_group_restaurants (
  id              uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  group_id        uuid NOT NULL REFERENCES user_restaurant_groups(id) ON DELETE CASCADE,
  restaurant_id   uuid NOT NULL REFERENCES restaurants(id) ON DELETE CASCADE,
  user_id         uuid NOT NULL,
  created_at      timestamptz DEFAULT now(),
  UNIQUE(group_id, restaurant_id)
);
CREATE INDEX IF NOT EXISTS idx_ugr_group ON user_group_restaurants(group_id);
ALTER TABLE user_group_restaurants ENABLE ROW LEVEL SECURITY;
CREATE POLICY "用户只能操作自己的分组店铺" ON user_group_restaurants FOR ALL USING (auth.uid() = user_id);
```

### 修改现有表

```sql

### 新增/修改路由

| 路由 | 方法 | 说明 |
|------|------|------|
| `/api/restaurants/avoid` | POST | 避雷店铺 |
| `/api/restaurants/unavoid` | POST | 取消避雷 |
| `/api/restaurants/avoided` | GET | 获取避雷列表 |
| `/api/restaurants/delete` | POST | 删除店铺（全局隐藏） |
| `/api/favorites/update-note` | POST | 更新收藏理由 |
| `/api/groups` | GET | 获取用户分组列表 |
| `/api/groups` | POST | 创建分组 |
| `/api/groups/{group_id}` | DELETE | 删除分组 |
| `/api/groups/{group_id}/restaurants` | POST | 添加店铺到分组 |
| `/api/groups/{group_id}/restaurants/{restaurant_id}` | DELETE | 从分组移除店铺 |
| `/api/authors/{author_id}/stats` | GET | 获取博主统计（餐厅数、粉丝数、城市数） |
| `/api/map/restaurants` | GET | **修改**：过滤已删除店铺，标记避雷店铺 |

## 四、各页面详细 UI 设计

### 4.1 MainTabView（修改）

Tab 结构（4个 Tab + 居中+按钮）：
- Tab 1: 地图 (`map.fill`)
- Tab 2: 占位（居中+按钮，弹出解析链接/手动添加选择面板）
- Tab 3: 收藏 (`bookmark.fill`) ← 合并原博主+收藏功能
- Tab 4: 我的 (`person.circle.fill`)
- Tab 5: 复核（仅管理员可见）

### 4.2 MapView（修改）

**右上角按钮组**（垂直排列）：
- 添加店铺按钮（橙色圆形，44pt，`plus` 图标）→ 弹出添加菜单
- 筛选展开/收起按钮（毛玻璃背景圆形，44pt）→ 切换筛选栏显隐

**筛选栏**（默认隐藏，点击展开时从顶部滑入）：
- 博主筛选：全部、"我的推荐"（使用用户真实头像）、各博主头像
- 店铺分组筛选：全部、收藏、避雷

**店铺标注点击**：从底部卡片改为 NavigationLink 跳转到 RestaurantDetailView 全屏页

### 4.3 FavoritesView（完全重写）

**页面结构**：
```
NavigationStack {
    ScrollView {

        //   - 已关注博主列表（博主头像+昵称，副标题"抖音达人"，点击→AuthorDetailView）

**关注列表行**：
- 圆形头像（44pt）+ 昵称 + 副标题（"抖音达人" 或 "我"）
- 点击自己 → RestaurantListView（店铺列表分组管理页）
- 点击博主 → AuthorDetailView（博主详情页）

**收藏店铺卡片**：
- 左侧：店铺图片（56pt 圆角）
- 中间：店铺名称 + 地址 + 分类标签；收藏理由（若有，灰色小字显示在名称下方）
- 右侧图标：💬留言图标（点击编辑收藏理由）+ 📍导航图标（打开地图导航）
- 左滑操作：
  - 橙色「收藏/取消收藏」按钮
  - 灰色「避雷」按钮
  - 红色「删除」按钮

### 4.4 AuthorDetailView（新增）

**页面说明**：从收藏页点击某个博主进入，展示博主完整信息和其推荐的所有店铺。

**布局**：
- 顶部信息卡片：博主头像（72pt 圆形）+ 昵称 + "抖音达人"标签 + 关注/取消关注按钮（胶囊形）
- 统计数据栏（三列）：餐厅数 | 粉丝数 | 城市数（灰色背景圆角卡片）
- 推荐店铺列表（LazyVStack）：复用店铺卡片行组件，支持左滑操作（收藏、避雷、删除）

**交互**：
- 关注状态切换：已关注→点击弹出 confirmationDialog 二次确认（"取消关注后，该博主推荐的店铺（已收藏店铺除外）将不再在地图上显示"）
- 未关注→点击直接关注
- 店铺行点击→跳转 RestaurantDetailView

### 4.5 RestaurantDetailView（新增，全屏页面）

**页面说明**：替代原地图底部卡片，提供店铺的完整信息和所有操作入口。从地图标注点击、收藏列表点击、博主详情店铺列表点击均可进入。

**布局**：
- 顶部封面图（全宽，高度 220pt，AsyncImage 加载店铺图片）
- 店铺名称 + 分类标签（CategoryBadge）
- 地址行（mappin.and.ellipse 图标 + 地址文字）
- 操作按钮网格（2列 × 3行 LazyVGrid）：
  - 导航（蓝色）→ 打开地图导航
  - 收藏/已收藏（橙色）→ 切换收藏状态
  - 分享（灰色）→ 占位，后续实现
  - 分组（紫色）→ 弹出分组选择 Sheet
  - 避雷/已避雷（红色/灰色）→ 切换避雷状态
  - 删除（红色）→ 弹出二次确认 Alert
- 收藏理由区域（若已收藏）：显示/编辑收藏理由
- 关联博主推荐区域：展示推荐该店铺的博主视频列表

**交互**：
- 删除二次确认："删除后该店铺将不再显示在地图和列表中"
- 避雷切换即时生效，地图标注样式同步变化

### 4.6 RestaurantListView（新增）

**页面说明**：从收藏页点击"自己"进入，管理用户所有店铺的分组视图。

**布局**：
- 系统预设分组（不可编辑/删除）：
  - 全部店铺（`square.grid.2x2`，蓝色）
  - 收藏的店铺（`bookmark.fill`，橙色）
  - 避雷的店铺（`exclamationmark.shield.fill`，红色）
- 自定义分组区域（Section "我的分组"）：
  - 每行：文件夹图标 + 分组名 + 店铺数量
  - 左滑删除分组

**交互**：
- 右上角 `folder.badge.plus` 按钮 → 弹出创建分组 Sheet（输入分组名）
- 点击任意分组行 → 进入 GroupDetailView（展示该分组内的店铺列表）

### 4.7 GroupDetailView（新增）

**页面说明**：展示某个分组内的店铺列表，支持添加/移除店铺。

**布局**：
- 店铺列表（复用店铺卡片行组件）
- 左滑可从分组中移除
- 右上角添加按钮 → 弹出搜索 Sheet，从已有店铺中选择添加到分组
