## 2026-03-08 - OpenClaw 命令找不到问题修复

### 问题描述
在新终端中执行 `openclaw` 命令时报错 `bash: openclaw: command not found`

### 原因分析
- 系统默认使用 bash 而非 zsh
- nvm 配置只在 ~/.bashrc 中
- ~/.bash_profile 没有加载 ~/.bashrc

### 解决方案
在 ~/.bash_profile 末尾添加加载 ~/.bashrc 的代码：

```bash
# 加载 .bashrc 以确保 nvm 等配置生效
if [ -f "$HOME/.bashrc" ]; then
    source "$HOME/.bashrc"
fi
```

### 修改的文件
- ~/.bash_profile - 添加加载 .bashrc 的配置
- ~/bin/openclaw - 创建符号链接

## 2026-03-08 - OpenClaw 命令找不到问题（第二次修复）

### 问题描述
在新终端中执行 `openclaw` 仍然报错 `command not found`

### 原因分析
- 可能是 shell 配置加载顺序问题
- nvm 在某些情况下未能正确加载

### 解决方案
1. 在 ~/bin/ 目录创建 openclaw 符号链接
2. 将 ~/bin 添加到 PATH

```bash
mkdir -p ~/bin
ln -sf /Users/xiangzy/.nvm/versions/node/v22.22.1/bin/openclaw ~/bin/openclaw
```

3. 在 ~/.bashrc 和 ~/.bash_profile 中添加 ~/bin 到 PATH

### 修改的文件
- ~/.bash_profile - 添加 ~/bin 到 PATH
- ~/.bashrc - 添加 ~/bin 到 PATH
- ~/bin/openclaw - 符号链接
