# 博主自动更新检测配置说明

## 功能概述

博主自动更新检测是 v2.4 新增的功能，可以自动发现已关注博主的新视频并解析入库，无需用户手动提交链接。

## 配置项说明

在 Railway 控制台的环境变量中添加以下配置：

### 基础配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `AUTO_UPDATE_ENABLED` | `false` | 是否启用自动更新检测。建议开发调试阶段设为 `false`，正式上线后再开启 |
| `AUTO_UPDATE_SCHEDULE_HOUR` | `2` | 每天定时检测的小时（0-23）。默认凌晨 2 点，此时段用户访问量最低 |
| `AUTO_UPDATE_MAX_AUTHORS_PER_RUN` | `50` | 每次定时任务最多处理多少个博主，防止一次处理太多导致费用失控 |
| `AUTO_UPDATE_TRIGGER_SECRET` | - | **必填**。触发密钥，用于验证 Railway Cron 请求来源 |

### 触发密钥生成

建议使用随机字符串作为触发密钥：

```bash
# 使用 OpenSSL 生成随机密钥
openssl rand -hex 32
```

### 调试阶段配置

**开发调试阶段**建议配置：

```bash
AUTO_UPDATE_ENABLED=false
DEBUG_MODE=true
DEBUG_MAX_VIDEOS=5
COMMENT_REPLY_DAILY_LIMIT=50
```

**正式上线阶段**建议配置：

```bash
AUTO_UPDATE_ENABLED=true
DEBUG_MODE=false
DEBUG_MAX_VIDEOS=30
COMMENT_REPLY_DAILY_LIMIT=100
AUTO_UPDATE_SCHEDULE_HOUR=2
AUTO_UPDATE_MAX_AUTHORS_PER_RUN=50
```

## Railway 部署配置

### 1. 主服务（端口 8000）

主服务处理用户请求，不需要特殊配置。

### 2. 调度器服务（端口 8001）

需要单独部署调度器服务来处理定时任务：

**Railway 部署步骤：**

1. 在 Railway 控制台添加新服务
2. 连接到同一个 GitHub 仓库
3. 设置构建命令：`pip install -r backend/requirements.txt`
4. 设置启动命令：`cd backend && python scheduler.py`
5. 配置环境变量（见上表）
6. **重要**：设置端口为 `8001`

### 3. Cron Job 配置

在 Railway 控制台为调度器服务添加 Cron Job：

1. 选择调度器服务
2. 添加 Cron Job
3. 配置 Cron 表达式：`0 2 * * *`（每天凌晨 2 点）
4. 配置请求：

```
Method: POST
URL: https://your-scheduler-service.railway.app/api/trigger-auto-update
Body: {"secret": "your-trigger-secret"}
Content-Type: application/json
```

## 本地开发测试

### 立即执行检测

```bash
cd backend
python scheduler.py --now
```

### 启动调度器 Web 服务

```bash
cd backend
python scheduler.py
```

服务将运行在 http://localhost:8001

### 手动触发检测

```bash
curl -X POST http://localhost:8001/api/trigger-auto-update \
  -H "Content-Type: application/json" \
  -d '{"secret": "your-trigger-secret"}'
```

### 测试单个博主

```bash
curl -X POST http://localhost:8001/api/trigger-single-author/author-uuid \
  -H "Content-Type: application/json" \
  -d '{"secret": "your-trigger-secret"}'
```

## 费用说明

自动更新检测会产生以下费用：

| 操作 | 接口 | 费用 |
|------|------|------|
| 获取博主视频列表 | `get-user-video-list/v3` | ¥0.1/次 |
| 视频详情扩展信息 | `get-video-detail/v2` | ¥0.1/次 |
| 视频评论 | `get-video-comment/v1` | ¥0.1/次 |

假设 100 个博主，其中 20% 有新视频，平均 1.5 条新视频：

- 每天约 100 次列表接口 = ¥10
- 每天约 30 次解析接口 = ¥3
- **每日合计约 ¥13**

## 自动关闭机制

为防止无效博主持续消耗费用，系统实现了自动关闭机制：

- 连续 **7 天**未检测到新美食视频 → 自动关闭该博主的自动更新检测
- 用户手动提交该博主的新视频且 AI 判断为美食类 → 重新激活自动更新

## 数据库字段

`authors` 表新增字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `auto_update_enabled` | boolean | 是否启用自动更新检测，默认 true |
| `last_update_check` | timestamptz | 上次执行自动检测的时间 |
| `no_new_food_video_days` | int | 连续未检测到新美食视频的天数 |

如需手动修改，可在 Supabase 控制台的 Table Editor 中操作。
