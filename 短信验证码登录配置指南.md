# 短信验证码登录配置指南

## 架构概览

```
iOS App (LoginView.swift)
    ↓
/api/auth/send-otp (发送验证码)
    ↓
阿里云短信 API
    ↓
用户手机收到短信
    ↓
用户输入验证码
    ↓
/api/auth/verify-otp (验证验证码)
    ↓
返回 user_id + access_token
    ↓
用户登录成功
```

## 配置步骤

### 1️⃣ 阿里云短信模板配置

**已完成**：你已在阿里云创建了短信模板

模板内容：
```
您的验证码：${code}，5分钟内有效，请勿泄露给他人。
```

场景说明：
```
用于"跟吃"App的用户登录/注册场景。用户在登录页面输入手机号后，点击"获取验证码"按钮，系统向该手机号发送短信验证码，用户输入验证码完成身份验证并登录。
```

### 2️⃣ 获取阿里云密钥

#### 第一步：获取短信模板 Code

1. 登录 [阿里云控制台](https://dysms.console.aliyun.com)
2. 左侧菜单 → **国内短信** → **模板管理**
3. 找到你的模板，复制 **模板 Code**（格式如 `SMS_123456789`）

#### 第二步：获取 AccessKey ID 和 AccessKey Secret

**方法一：通过阿里云账户管理**

1. 登录 [阿里云控制台](https://www.aliyun.com)
2. 右上角点击你的 **账户头像** → 下拉菜单选择 **AccessKey 管理**
3. 进入 AccessKey 管理页面后，你会看到两个选项卡：
   - **当前用户 AccessKey**（推荐用这个）
   - **子用户 AccessKey**

4. 在 **当前用户 AccessKey** 标签下：
   - 如果已有 AccessKey，直接点击 **显示** 查看 Secret
   - 如果没有，点击 **创建 AccessKey** 按钮创建新的

5. 复制以下两个值：
   - **AccessKey ID**（格式如 `LTAI5tXXXXXXXXXX`）
   - **AccessKey Secret**（格式如 `XXXXXXXXXXXXXX`）

> ⚠️ **重要提示**：
> - AccessKey Secret 只会显示一次，请妥善保管
> - 不要在代码中硬编码，只在 `.env` 文件中配置
> - 定期轮换 AccessKey 以提升安全性

### 3️⃣ 配置本地环境变量

**文件：** `backend/.env`

```bash
# 短信验证码配置
ALIYUN_ACCESS_KEY_ID=your_access_key_id
ALIYUN_ACCESS_KEY_SECRET=your_access_key_secret
SMS_SIGN_NAME=跟吃
SMS_TEMPLATE_CODE=your_template_code
```

**替换说明：**
- `your_access_key_id` → 你的 AccessKey ID
- `your_access_key_secret` → 你的 AccessKey Secret
- `your_template_code` → 你的短信模板 Code

### 4️⃣ 配置 Railway 环境变量

1. 登录 [Railway 控制台](https://railway.app)
2. 选择你的项目 → **Variables**
3. 添加以下环境变量：

| 变量名 | 值 |
|--------|-----|
| `ALIYUN_ACCESS_KEY_ID` | 你的 AccessKey ID |
| `ALIYUN_ACCESS_KEY_SECRET` | 你的 AccessKey Secret |
| `SMS_SIGN_NAME` | 跟吃 |
| `SMS_TEMPLATE_CODE` | 你的模板 Code |

## 常见问题

### Q: 验证码一直收不到？
**A:** 
1. 检查阿里云短信模板是否已审核通过
2. 检查手机号格式是否正确（需包含 +86）
3. 检查 AccessKey 是否正确
4. 查看阿里云控制台的短信发送日志

### Q: 如何修改验证码有效期？
**A:** 编辑 `backend/sms_service.py`，修改 `OTP_EXPIRE_SECONDS` 常量（默认 300 秒 = 5 分钟）

### Q: 如何修改验证码长度？
**A:** 编辑 `backend/sms_service.py`，修改 `generate_otp()` 函数中的随机数范围

### Q: 生产环境如何存储验证码？
**A:** 当前使用内存存储，生产环境建议改为 Redis：
```python
# 改为 Redis 存储
import redis
r = redis.Redis(host='localhost', port=6379, db=0)
r.setex(f"otp:{phone}", OTP_EXPIRE_SECONDS, code)
```
